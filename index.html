Long way to get here!

经过两天的时间，终于到达了这里，拥有了自己的博客，心情怎么能不小愉悦小鸡冻呢？在这短时间里， 遇到了不少有趣的问题，甚至有的是很大只的“拦路虎”。不过这都是小小的磨练。趁新鲜出炉，先总结一下吧。

1. yml文件的空格

  • 问题现象：hexo d 部署时，没有报错，而且很快就通过了，但是访问wulakuer.github.io， 却返回404的错误。

  • 定位问题： 回忆一下，本地可以顺利访问，github上的代码也米有问题，那就只能是部署的问题。与部署密切相关的就是配置文件：_config.yml。其中包含了部署信息。

  • 问题解决：

    deploy:
    type: git
    repository: git@github.com:wulakuer/wulakuer.github.io.git
    branch: master

    ：后面要有一个空格。

2. 强制重新deploy

  • 问题现象： 更换主题之后，本地访问localhost:4000正常，上传到github上的代码与本地也一致，但是deploy之后，访问wulakuer.github.io 主题没有任何变化，如果加了新的文章，做deploy,wulakuer.github.io也还是没有任何变化，与之前一样。而hexo deploy没有报错，并且输出了正确的log
  • 定位问题：这说明deploy并没有成功。
  • 问题解决：在本地删除.deploy_git文件夹，重新执行 hexo clean hexo g hexo d, 做到强制重新deploy

感谢漩涡的博客的文章http://blog.csdn.net/wx_962464/article/details/4478692 漩涡的博客中都提到了这两个问题和解决办法，非常实用。
